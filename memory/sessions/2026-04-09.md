## Sessão 4 (madrugada 10/04)

### Extração custos Full (Fulfillment) por SKU
- Investigação API ML: endpoints `/fulfillment/fees`, `/items/{id}/costs`, `/billing`, `/charges` — todos 404
- Descoberta: custos Full estão no `charges_details` do pagamento Mercado Pago (`/v1/payments/{id}`)
  - `shp_fulfillment` (type=shipping): custo envio Full (manuseio + armazenagem combinados)
  - `ml_sale_fee` (type=fee): comissão ML
  - `mp_processing_fee` (type=fee): taxa processamento MP
- ML **não separa manuseio de armazenagem** — é um valor único no `shp_fulfillment`
- Script v1 morreu em 600/2852 (possível rate limit). Script v2 com: nohup, progress file incremental, sleep 0.8s, retry com backoff
- Script v2 processou 2866 pedidos (60 dias), 57 SKUs, 0 erros
- Output: CSV (`~/Downloads/ml-full-costs-by-sku.csv`) + JSON (`/tmp/ml-full-costs-by-sku.json` na VPS)
- Padrão: cross_docking = Full R$0 (vendedor envia), fulfillment = custo real
- Piores custos Full: KIT6S100 R$27,94, KIT4YW1520 R$25,50, EMB01T R$18,21

### Esclarecimentos planilha MELI
- Confirmado que `list_cost` (Col R) é frete de entrega, NÃO inclui Full
- Full (Col N) é separado: manuseio + armazenagem no CD do ML
- Col K (CUSTO FIXO) zerada corretamente — já inclusa no list_cost desde março 2026

## Decisões
- Custos Full via Mercado Pago `charges_details` (único endpoint que retorna)
- Rate limit conservador: sleep 0.8s entre chamadas MP, 0.5s entre shipments
