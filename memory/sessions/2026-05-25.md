## Sessão noturna — Lapidação L06 (linguagem natural para Yasmin)

Sessão longa de refinamento de comunicação no pipeline DSA ML. Máquina analítica (L01-L05) intacta; foco em simplificar a saída final no Slack.

### O que foi feito

- **GMV → Faturamento** — substituição linguística total. Yasmin não usa "GMV" no dia a dia. Regra ativa em L05 (não passa adiante) + L06 (tabela de tradução) + L07 (bloqueio Maior se escapar).
- **`ADS share` → expressões naturais** — "ADS respondeu por X%", "fatia do ADS no faturamento". L05 traduz nos campos `analise_final_condensada`/`prioridades_condensadas`/`memoria_para_amanha`. L07 detecta jargão técnico como Menor se aparecer 3+ vezes.
- **4 termos naturais para nível de qualidade do anúncio** (campo `health` da API ML, escala 0-1):
  - `0,99-1,00` → **excelente** (= Profissional ML)
  - `0,85-0,98` → **bom** (= Padrão superior ML)
  - `0,70-0,84` → **regular** (= Padrão inferior ML)
  - `0,00-0,69` → **preocupante** (= Básico ML)
  - `null` → "sem nível calculado pelo ML"
  - L01-L05 continua usando número `health` para cálculos; só L06 traduz para termo natural.
  - **Pontuação numérica removida do Slack** — Yasmin não precisa do valor exato; basta o nome da faixa.
  - **Duplicação de "nível" proibida** — formato natural: "qualidade do anúncio em regular" / "qualidade dos dois anúncios em bom" / "anúncio com qualidade preocupante".
- **Princípios de comunicação simples (Pedro 2026-05-25)** — seção completa adicionada ao prompt L06:
  - **Princípio rector:** "Yasmin não precisa saber como a análise foi feita. Ela precisa saber: o que aconteceu, o que fazer, e por que importa."
  - **Tabela de tradução** de ~15 termos técnicos (ETA, breakdown, runway, elegibilidade, share, threshold, ratio, snapshot, prospectivo, variável confundidora, erosão de exposição, cluster, mix, patamar, etc.).
  - **Regras de estilo:** frase ≤25 palavras, verbo no lugar de substantivo abstrato, sujeito concreto, ordem direta.
  - **4 exemplos antes/depois** calibrando o tom.
  - **Regra dura:** preservar números, classificações (fato/hipótese/risco latente), nomes próprios e vocabulário ML obrigatório. Simplificar APENAS frases longas, jargão técnico não-ML, estruturas pesadas.
- L07 (QA Gate) atualizada: bloqueio Menor se 3+ termos da lista aparecerem sem tradução; bloqueio Maior para "health" solto, "GMV", pontuação numérica, duplicação de "nível", termos oficiais ML.
- **Validação empírica** — múltiplas execuções iterativas. Run final (`08-slack-sent.txt` de 24/05) saiu APROVADO COM RESSALVA com 2 menores residuais (fidelidade fina à L05, não comunicação). Trecho representativo: *"Campanha eficiente em fase de leitura não se toca."* / *"vendendo cada vez mais por um anúncio que o ML vem reduzindo em aparecimento orgânico."*

### Decisões
