## [09/04] Refatoração planilha MELI — Comissões, frete e devoluções reais

**Contexto:** Planilha de estoque usava comissão 12,5%, frete R$0 para produtos <R$79, e taxa de devolução fixa 4% — todas estimativas incorretas. Margem real desconhecida.
**Decisão:** Corrigir 3 colunas da aba MELI (45 SKUs ativos):
- Col J (COMISSÃO): fórmula `=G*11%` (Clássico confirmado para todos via API)
- Col R (FRETE): valor real por SKU (média últimas 30 vendas, 1887 pedidos analisados) usando `=CENTS/100`
- Col K (CUSTO FIXO): zerado (já incluso no frete)
- Col O (DEVOLUÇÕES): taxa real por SKU do arquivo xlsx (27 SKUs com dados, 18 em default 4%)
**Resultado real:** 20 SKUs OK (≥5%), 3 ALERTA (0-5%), 22 PREJUÍZO (<0%)
**Pior caso:** CNCOL-80 PCS (-22%), KIT2YW320 (-21%), SPC011 (-19%), EMB01T (-19%)
**Aprendizado:** Google Sheets PT-BR rejeita ponto decimal em fórmulas. Usar `=G*X%` ou `=G*CENTS/1000`.
**Owner:** Pedro (via Claude Code + SSH)
**Status:** ✅ Aplicado — planilha recalculada com dados reais

## [10/04] Extração custos Full (Fulfillment) por SKU via Mercado Pago

**Contexto:** Custos de Full não estão disponíveis na API do ML diretamente. Endpoints `/fulfillment/fees`, `/items/{id}/costs`, `/billing`, `/charges` — todos retornam 404. Precisávamos de dados reais por SKU para precificação.
**Descoberta:** Os custos de Full aparecem no `charges_details` do pagamento no Mercado Pago (`GET /v1/payments/{id}`), como charge com `name: "shp_fulfillment"` e `type: "shipping"`. O ML **não separa manuseio de armazenagem** — é um valor único.
**Execução:** Script Python processou 2866 pedidos (60 dias), extraindo de cada pagamento: `shp_fulfillment`, `ml_sale_fee`, `mp_processing_fee`. Agrupado por SKU com média/min/max.
**Output:** CSV + JSON com 57 SKUs em `/tmp/ml-full-costs-by-sku.json` (VPS) e `~/Downloads/ml-full-costs-by-sku.csv` (local).
**Dados-chave:**
- cross_docking = Full R$0 (vendedor envia direto)
- fulfillment = custo real (manuseio+armazenagem combinados)
- Piores: RED01B 84,3% retido, IMB501P_T 48,7%, 914C_BAP 46,1%
**Owner:** Pedro (via Claude Code + SSH)
**Status:** ✅ Dados extraídos — pendente integração na planilha Col N
