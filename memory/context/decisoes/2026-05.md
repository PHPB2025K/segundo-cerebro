## [02/05] PEP 668 — sempre venv para skills Python no macOS Homebrew

Python 3.14 instalado via Homebrew bloqueia `pip install` global (PEP 668). Skills Python externas (clones de repos como `video-use`) devem usar **venv isolado** via `uv sync`.

**Decisão de execução:**
- `brew install uv` se não existir.
- `uv sync` no repo para criar `.venv/` automático.
- Invocar helpers SEMPRE via `<repo>/.venv/bin/python`, nunca via `python` do sistema.
- Memória local persistente registrada para garantir que sessões futuras do Claude Code apliquem a regra ao usar a skill `video-use`.

## [02/05] Permissão restrita de API key ElevenLabs

API key do ElevenLabs criada para `video-use` deve ser **restrita ao endpoint `Speech to Text → Access`**. Todo o resto = `No Access`. Princípio do menor privilégio.

**Justificativa:** a skill chama um único endpoint (`https://api.elevenlabs.io/v1/speech-to-text` em `helpers/transcribe.py:30`). Permissões adicionais aumentariam superfície de risco em caso de vazamento sem benefício.

**Limite mensal:** 30k chars sugerido (~Starter $5/mês) como trava de segurança.

**Sanity check de key restrita:** `POST /v1/speech-to-text` sem payload deve retornar HTTP 422 (auth OK, payload inválido). NÃO usar `/v1/user` para validar (exigiria permissão `Administration`).

## [01/05] DRE profissional — estrutura clássica obrigatória

Pedro rejeitou versões de DRE que, apesar de números corretos, pareciam apuração simplificada de marketplace. A referência obrigatória é a planilha `02_DRE_GB_PROCESSADO.xlsx`.

**Decisão:** toda DRE/P&L mensal da GB deve seguir estrutura clássica completa:
Receita Operacional Bruta → Deduções → Receita Operacional Líquida → CMV → Lucro Bruto → Margem Bruta → Despesas Operacionais → EBITDA → Margem EBITDA → Depreciação/Amortização → EBIT → Margem Operacional → Resultado Financeiro → LAIR → Provisões Tributárias → Lucro Líquido → Margem Líquida.

Se dados faltarem, manter a linha e marcar lacuna; não remover seção para simplificar.

## [01/05] Fiscal — reconciliação por SKU/componente antes de emitir

**Decisão:** antes de emitir NF interna mensal, o Fisco deve reconciliar:
- estoque fiscal inicial Matriz/Filial pós-mês anterior;
- entradas e transferências do mês por container;
- SKU fiscal/Bling vs SKU comercial/marketplace;
- fator de conversão de caixa/conjunto/unidade;
- decomposição de kits;
- vendas do mês por CNPJ;
- limite fiscal disponível por estabelecimento.

NF não pode ser gerada apenas pelo rateio das vendas.
