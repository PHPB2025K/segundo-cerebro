## [05/05] Canggu — root cause da regressão dupla (filtro Plataforma sumiu + Ana parou de classificar origem)

**Sintoma reportado pelo Pedro:** acessou `https://canguu-sigma.vercel.app/conversations` e (1) viu apenas 3 filtros (Status, Categoria, Sentimento) — o filtro de Plataforma implementado no commit `b2c6d0f` em 30/04 não estava visível; (2) percebeu que a Ana não estava mais mandando a mensagem numerada de classificação de origem nas conversas novas no WhatsApp.

**Diagnóstico:**
- **Filtro de Plataforma:** código presente no bundle deployado. `curl https://canguu-sigma.vercel.app/assets/index-B57kBlxQ.js | grep -E "Plataforma|Site Budamix"` confirmou que o JS em produção contém as strings. Componente `ConversationList.tsx:95-106` renderiza o `<Select>` incondicionalmente, sem feature flag. **Cache CDN do Vercel estava com `age: 393877s` (4.5 dias)** desde o último deploy em 30/04 18:09 UTC. Não era regressão de código.
- **Origin poll:** edge function `webhook-whatsapp` versão 28 deployada em **30/04 13:29 BRT**. Mas o commit `604b2e3` ("numbered-text origin poll + null source default") é de **30/04 17:28 BRT**, ~4h depois. Em produção rodava a versão anterior (`b2c6d0f`) que usava `sendList()` — Evolution + Baileys droppam silenciosamente esse tipo, ou mostram "aguardando mensagem" no celular do cliente. Numbered-text só foi pra produção agora.

**Resolução em 05/05 ~08:30 BRT:**
1. `supabase functions deploy webhook-whatsapp --project-ref jpacmloqsfiebvagfomt` → versão 29, com numbered-text origin poll.
2. `vercel redeploy <last-prod-deploy-id> --target=production` → cache CDN invalidado (`age: 0`, `last-modified` atualizado pra 05/05).

**Lição operacional:** edge functions Supabase **não auto-deployam** via push pro GitHub. Cada commit que toca `supabase/functions/*` precisa de `supabase functions deploy <name>` manual, **um por função**. Isso cria risco constante de drift entre repo (single source of truth declarada) e produção. Esse é o segundo evento documentado de drift desse tipo (primeiro foi tangencial na maratona de 30/04).

**Mitigação proposta para o roadmap Canggu (B5 — CI/CD):** GitHub Action no repo `PHPB2025K/canguu` que rode `supabase functions deploy --project-ref <ref>` automático no push pra main, idealmente filtrando por path (`supabase/functions/**`). Resolve a classe inteira de drift e remove necessidade de cada um lembrar de deployar manualmente.

**Owner:** CC local (diagnóstico + redeploy).

4. Cert Let's Encrypt emitido pós-restart; `acme.json` registra `main=ponto.budamix.com.br has_cert=True`
5. Validação interna (curl --resolve 127.0.0.1:443) e externa (curl --resolve IP:443) ambas retornaram HTTP/2 200

**Estado:**
- DNS público propagado em Google/Cloudflare/Quad9

- `https://ponto.budamix.com.br` — oficial

**Lição aprendida:** quando se ajusta router do Traefik pra novo Host antes do DNS existir publicamente, o ACME tenta emitir e cacheia o erro com backoff. Workflow correto é: criar DNS → esperar propagação → ajustar Traefik → reload. Caso a ordem seja invertida (como aqui), basta restartar o Traefik depois do DNS propagar pra forçar nova tentativa.

## [02/05] PEP 668 — sempre venv para skills Python no macOS Homebrew

Python 3.14 instalado via Homebrew bloqueia `pip install` global (PEP 668). Skills Python externas (clones de repos como `video-use`) devem usar **venv isolado** via `uv sync`.

**Decisão de execução:**
- `brew install uv` se não existir.
- `uv sync` no repo para criar `.venv/` automático.
- Invocar helpers SEMPRE via `<repo>/.venv/bin/python`, nunca via `python` do sistema.
- Memória local persistente registrada para garantir que sessões futuras do Claude Code apliquem a regra ao usar a skill `video-use`.

## [02/05] Permissão restrita de API key ElevenLabs

API key do ElevenLabs criada para `video-use` deve ser **restrita ao endpoint `Speech to Text → Access`**. Todo o resto = `No Access`. Princípio do menor privilégio.

**Justificativa:** a skill chama um único endpoint (`https://api.elevenlabs.io/v1/speech-to-text` em `helpers/transcribe.py:30`). Permissões adicionais aumentariam superfície de risco em caso de vazamento sem benefício.

**Limite mensal:** 30k chars sugerido (~Starter $5/mês) como trava de segurança.

**Sanity check de key restrita:** `POST /v1/speech-to-text` sem payload deve retornar HTTP 422 (auth OK, payload inválido). NÃO usar `/v1/user` para validar (exigiria permissão `Administration`).
