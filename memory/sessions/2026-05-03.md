## Amazon Ads — framework aprovado pelo Pedro

Pedro corrigiu a lógica estratégica da análise: Amazon Ads não pode ser tratada só como redução de ACoS. Toda revisão por grupo precisa conter 3 blocos:

1. **Cortar desperdício**: reduzir/pausar bleeders, negatives e termos ruins.
2. **Promover winners validados**: mover winners de Auto/Broad para Performance Exact, com negativa na origem quando fizer sentido.
3. **Escalar vendas**: subir bid de winners Exact já comprovadas para aumentar volume mantendo controle de ACoS/ROAS.

Regra operacional reforçada: **não escalar keyword recém-promovida para Exact no mesmo momento**. Winner na origem não é automaticamente winner na Performance; precisa acumular 3–7 dias de histórico próprio.

### Limitações técnicas confirmadas

Paper Shaders (Mesh Gradient, Grain, Liquid Metal) **não funcionam via MCP**. Testes com custom elements, `<MeshGradient>`, props CSS e SVG noise foram rasterizados como Frame genérico/cinza. Paper Shaders é lib React `@paper-design/shaders-react`, sem API CSS/HTML acessível via MCP.

Padrões aprendidos:

- `paper-asset:///path/absoluto` funciona, mas precisa de `width` e `height` explícitos; `width: auto` quebra/invisibiliza logo.
- Paper rasteriza custom element/CSS desconhecido silenciosamente.
- `duplicate_nodes` cross-artboard funciona bem para rodapés/logos/conteúdo repetido.
- Variáveis CSS em `<style>` no `write_html` não persistem entre chamadas; valores precisam ser literais.
- Logos devem usar dimensões explícitas baseadas no aspect ratio real do arquivo, não no quadrado wrapper.
