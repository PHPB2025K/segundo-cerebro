### Refatoração crítica do tracking de custos

Pedro questionou se `/costs` era custo real ou estimativa. Investigação:
- **Anthropic** estava marcada como subscription, deveria ser `api_billing` (Pedro paga via API)
- **GPT-5.5/5.4/5.1-mini** ausentes do pricing → fallback errado contava cada token como Haiku 3.5
- **Crons OpenClaw**: investigação completa confirmou 100% via `openai-codex` (subscription Pro 5x). 30 crons habilitados — zero usam Anthropic.

Correções aplicadas:
- `pricing.ts` reescrito: Anthropic → `api_billing` real, GPT-5.x → `subscription` (`realCostMultiplier=0`), Gemini → `external` real
- Migration `usage-tracking.db`: adicionadas colunas `billing_type`, `real_cost`, `equivalent_cost`. **4628 snapshots recalculados**.
  - Custo REAL total histórico: **$282.57** (Anthropic + Gemini)
  - Equivalente API se tudo fosse pay-per-token: **$2994.89**
  - Economia da subscription Pro 5x: **$2712.32**
- Por mês:
  - Maio: $0 API real (todo uso foi GPT-subscription) + $1241.81 equivalente coberto
  - Abril: $274.84 Anthropic API real + $1470.51 equivalente coberto
  - Março: $7.66 Anthropic API + parcial
- API `/api/costs/budget` GET/PUT aceita `subscriptionMonthly` separado
- Página: 2 cards novos ("Subscription mensal" editável $200 default; "Economia da subscription" verde). KPIs renomeados pra "API hoje/mês/média diária/projeção EOM". Budget só de API. Alerta 80%/100% só sobre real.
