## Decisões

- **Princípio operacional confirmado:** todo node do vault deve estar conectado semanticamente. Stub é dívida documental aceitável (alguém referenciou heading que ainda não foi escrito); node humano isolado é defeito.
- **Bloco 2 completo (descarte):** os `_RELATORIO_*` foram deletados ao invés de conectados — eram artefatos de scripts antigos sem valor canônico (dados já estão em `projects/{nome}.md`).
- **Bloco 3 completo (160 nodes):** corrigido em massa via script Python com templates por categoria + detecção semântica via regex. Trade-off aceito: wikilinks "âncora" padronizados (IDENTITY/SOUL do agente) em troca de ganho de escala.
- **Validação prévia de targets é mandatória:** próxima geração de scripts de auditoria deve sempre verificar existência dos alvos antes de inserir wikilinks. Aprendizado registrado em [[knowledge/concepts/kg-wikilinks-auditoria-metodo]].
- **Rebuild completo do KG (DB nuke + reindex)** é a única forma de purgar stubs zumbis — incremental não remove nodes mortos.
