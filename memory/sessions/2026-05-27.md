## Watchdog GitHub Sync — alarme de commits stuck + fix raiz — 22:30 BRT

Alarme do Watchdog GitHub Sync do Pedro disparou avisando que o vault local (VPS) estava à frente do GitHub. Investigação SSH no `/root/segundo-cerebro` confirmou: 3 commits manuais do Kobe (`Registra atualização de custos...`, `Registra desconto...`, `Registra taxas reais...`) presos há ~2h sem push.

### Causa raiz

Bug em `meta/scripts/safe-write.sh` modo `commit`: o early exit quando `git status --porcelain` está vazio NÃO verificava se havia commits locais não-pushados antes de sair. Cron `sync-memory-to-vault.sh` (a cada 15min) chamava `safe-write.sh commit "kobe-auto: ..."` → script via workspace limpo → `do_pull` + exit 0 → commits stuck ficavam presos até alguém modificar outro arquivo e disparar o branch "commit + push" do script.

Kobe respondeu via Pedro confirmando que era ele mesmo (main session) comitando inline com `git commit` direto em vez de chamar `safe-write.sh commit`. Sem motivo técnico — só legado operacional.

### O que foi feito

1. **Push manual imediato na VPS** (`3d39baf..e78173f`) — alarme resolvido em segundos
2. **Patch defensivo em `safe-write.sh`** (commit `c5102e0`) — detecta `git log origin/main..HEAD` mesmo com workspace limpo e pusha. Validado com cenário sintético na VPS: criou commit stuck via `git commit` direto, rodou `safe-write.sh commit` (msg irrelevante), script detectou e pushou.
3. **Regra operacional no `openclaw/agents/kobe/SOUL.md`** (commit `fe349d1f`) — nova seção "Escrita no vault" entre "Comunicação com Pedro" e "Never dos (absolutos)". Texto: toda escrita no vault DEVE passar por `safe-write.sh commit`; nunca `git commit` direto. Justificativa + referência ao incidente + códigos de saída.
4. **Nota de topo em `skills/obsidian-vault-manager/SKILL.md`** (mesmo commit) — desambigua "se vault for segundo-cerebro, NÃO usar `git commit` cru — os exemplos abaixo são pra vaults Obsidian genéricos de outros usuários".

### Decisões

→ **Toda escrita no `segundo-cerebro` por qualquer máquina ou agente DEVE passar por `safe-write.sh commit`.** Sem exceção operacional. `git commit` direto só em diagnóstico controlado autorizado explicitamente pelo Pedro. Regra agora carregada automaticamente no SOUL do Kobe.
→ **Defesa em profundidade no `safe-write.sh`:** mesmo que algum processo viole a regra e comite direto, o próximo ciclo do cron de autosave (cada 15min) faz catch-up via `push origin main` automático. Sintoma "commits stuck" não deve mais alarmar o Watchdog.
