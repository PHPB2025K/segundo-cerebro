## O que foi feito

### Diagnóstico honesto do Knowledge Graph
- Relatório em 5 seções sobre o estado de uso do KG (`~/.local/share/knowledge-graph/kg.db`)
- **Conclusão principal:** KG está instalado e conectado (via MCP), mas foi **usado ZERO vezes** durante a sessão inteira de 19/04 (4 grupos BidSpark-3 + compute-results fix + 3 skill updates). Toda a carga de contexto veio de `cat`/`grep` direto ou queries SQL ao Supabase.
- Causa diagnosticada: o `/cerebro` da skill usa Bash direto, e Claude Code esquece que KG existe porque não tem gatilho explícito no fluxo mental.
- Teste funcional (kg_search "auditoria de capas de produtos budamix ecommerce") retornou 20 resultados com score top 0.55 — relevância morna, e a sessão 2026-04-19 recém-criada NEM APARECEU (stale data porque o kg_index só rodou DEPOIS da sessão 2 ser adicionada).

### Implementadas 2 mudanças cirúrgicas (Pedro decidiu skip a variante /cerebro <tópico>)

1. **kg_index automático no /cerebro** ([`~/.claude/skills/cerebro/SKILL.md`](~/.claude/skills/cerebro/SKILL.md) linha ~86, Fase 0 nova no Modo Briefing)
   - Fire-and-forget silencioso: se MCP offline ou timeout, seguir sem bloquear briefing
   - Custo 2-5s, elimina stale data que o teste de hoje expôs
   - Fallback CLI documentado mas não default

2. **Gatilhos automáticos + freshness check no CLAUDE.md global** ([`~/.claude/CLAUDE.md`](~/.claude/CLAUDE.md) linhas 182-218, 2 subseções novas dentro de "Estratégia de Busca")
   - **Gatilhos linguísticos:** palavras que disparam `kg_search` antes de Grep/Glob/Read — "lembrar", "onde está", "tudo sobre", "contexto de", "relacionado a", "já falei sobre", "aquela nota sobre", "me conta sobre", "o que sei de", "resumo de"
   - **Anti-gatilhos explícitos** para prevenir falso-positivo: path específico, nome exato de função/classe/arquivo, pedido de Grep, tarefa de código ativo
   - **Freshness check:** `stat` no `kg.db` mtime; se >4h stale → `kg_index` antes do primeiro `kg_search`. Exceção: se `/cerebro` rodou na sessão (agora roda kg_index auto), pular check.

## Decisões

- **Aplicar só #1 e #3** da proposta original, skip #2 (variante `/cerebro <tópico>`). Motivo: o pull-on-demand do `/cerebro` já tem comando `busca [query]` que faz o mesmo; duplicar via atalho paramétrico seria conforto, não necessidade. Se em 3-5 dias de uso os gatilhos não forem suficientes, reavaliar.
- **Não commitar `~/.claude/` agora.** Arquivos de configuração pessoal; Pedro decide quando versionar.
- **Risco de falso-positivo aceitável** dos gatilhos: lista foi propositalmente narrow (não inclui verbos genéricos tipo "o que"/"me fala"). Anti-gatilhos cobrem os casos de overreach.
