## [03/06] Klap — dedup massivo de product_images (424 → 169)

**Contexto:** carrosséis das PDPs estavam carregando até 21 fotos por produto, várias visualmente idênticas (mesma foto Amazon SP-API em 3 resoluções: large/medium/`_SL75_` thumbnail).
**Decisão:** rodar pipeline de dedup em duas passadas — (1) URL base_id Amazon ignorando sufixo de tamanho removeu 108 thumbnails; (2) perceptual hash (pHash, threshold ≤ 5 Hamming) detectou outras 147 duplicatas entre base_ids Amazon diferentes (`7XXXX` vs `4XXXX` = mesma foto, IDs distintos).
**Resultado:** 255 fotos removidas, cada produto fica com 5–8 fotos únicas no carrossel. Mudança imediata em produção (sem deploy — só Supabase).
**Owner:** Pedro + Claude
**Status:** concluído

## [04/06] Regra de densidade de 4 elementos para risco silencioso por gap estrutural

**Contexto:** primeiro smoke do Shopee (01/06) produziu insight raso em gap estrutural: *"A Saúde da Loja está totalmente indisponível e Shopee Ads inacessível"* — descrevia o gap sem dimensionar nem propor ação. Pedro disse: "Utilize como base os prompts específicos que foram usados para as análises do Mercado Livre como a principal referência. Aquele nível de densidade é o que eu quero."

**Decisão:** sempre que um insight tocar um "risco silencioso por gap estrutural" (ex: dado de saúde indisponível, métrica não fetchável), o output deve conter **no mínimo 4 elementos**:

1. **Estimativa derivada** (ordem de grandeza a partir de proxy disponível — não deixar como buraco)
2. **Threshold com referência** (qual valor seria sinal de alerta, citando fonte: benchmark interno, política Shopee, histórico próprio)
3. **Janela D+N** (em quantos dias o gap precisa ser resolvido antes de virar perda mensurável)
4. **Fonte humana alternativa** (canal manual de validação enquanto o gap técnico não é resolvido: ex. "checar manualmente no Seller Center > Saúde da Loja semana atual")

**Aplicação:** regra aplicada em L01 (após Lente 7), L05 (sub-regra no Padrão F) e L06 (regra "Densidade obrigatória"). L02/L03/L04 ainda pendentes (ver `pendencias.md`).

**Por que importa:** sem essa estrutura, um insight de gap vira "tem buraco aqui" — descritivo e inacionável. Com os 4 elementos, vira hipótese dimensionada + decisão dentro de uma janela + plano B humano enquanto não tem dado técnico.

**Onde está documentada:** dentro dos próprios prompts (L01 linha ~570, L05 Padrão F, L06 seção Densidade obrigatória). README v4.0/shopee menciona como decisão de arquitetura.

**Owner:** Pedro + Claude
**Status:** implementada parcialmente, expansão pendente

- Quirk: `/me/accounts` retornou `[]` mesmo com permissões corretas. Workaround: GET direto na página com user long-lived retorna o page token

**Edições aplicadas:**
