# Auditoria Forense Canggu — 2026-04-22

## Contexto

Auditoria forense READ-ONLY do sistema Canggu em preparação para refatoração.

sem modificar banco, repo ou VPS.

## Achados principais

Ver [[debitos-tecnicos]] para matriz completa de 24 achados.

### Críticos (🔴)

#### Achado 1 — Service role JWT exposto
Hardcoded em 3 arquivos versionados (desde 2026-03-06): 2 migrations + 1 [[n8n-workflows|workflow N8N]]. Bypass RLS + acesso root ao banco. Ver [[decisoes#ADR-001]] para plano de mitigação.

#### Achado 2 — Ghost function `test-search`
[[edge-functions#Deletar|test-search]] deployed sem código no repo (comentário `// DELETE after testing`), `verify_jwt=false`, 8 calls OpenAI por invocação. Custo potencial de abuso ~US$7k/dia. Ver [[decisoes#ADR-003]].

#### Achado 3 — Zero monitoria ativa
0 cron jobs, 0 DB webhooks, Health Check N8N simplificado de 11 para 2 nodes no live (SMTP desassociado). Falhas silenciosas indetectáveis exceto por reclamação externa. Ver [[debitos-tecnicos#B2]].

### Cross-layer novos (só apareceram integrando Fase 1+2)

- **Pipeline WhatsApp duplicado** (edge function + N8N implementam o mesmo fluxo) — resolvido em [[decisoes#ADR-007]]
- **Pipeline ML não migrou para `base_products`** — pendente em [[decisoes#ADR-006]]
- **3ª ocorrência do service_role hardcoded** em migration untracked (working copy dirty — não estava no grep inicial)

## Pontos fortes identificados (preservar)

- **11 FKs no schema** — integridade referencial intencional (contraste com BC que tem 0)
- **TypeScript strict + `noUncheckedIndexedAccess`** — config restritiva rara em projetos deste estágio
- **pgvector 0.8.0 em uso real** (5 tabelas com vector columns) alimentando RAG via triggers
- **13 migrations evolutivas coerentes** — zero objetos fantasma, zero drift óbvio
- **`_shared/` modularizado** em 12 módulos bem separados por responsabilidade
- **Debounce 12s** no pipeline WhatsApp — pattern que evita múltiplas respostas a bursts
- **Escalation pipeline** estruturado (`checkEscalation` → `human_agent`)
