**Ações executadas (CC local + Pedro no painel DNS):**

   - Primeira tentativa: Pedro digitou `pontocerto` por engano → orientado a corrigir pra `ponto`
   - Segunda tentativa: criou mas não clicou em `SALVAR ALTERAÇÕES` → registro não persistiu
   - Terceira tentativa: salvou de fato. Ficou também `pontocerto.budamix.com.br` órfão na zona (inofensivo, pode deletar pra higiene)
2. DNS propagou em todos os resolvers públicos em <2min (Google 8.8.8.8, Cloudflare 1.1.1.1, Quad9)
3. **Cert SSL Let's Encrypt falhou na primeira tentativa do Traefik** (12:39:46 UTC, antes do DNS propagar) — erro `400 :: DNS problem: NXDOMAIN looking up A for ponto.budamix.com.br`
4. Traefik não retentaria automaticamente até o próximo ciclo. CC local fez `docker restart traefik-traefik-1` na VPS (autorizado pelo Pedro — causa 5-10s de indisponibilidade nos outros serviços: central, estoque, import, pontocerto)

6. Validação interna: `curl --resolve ponto.budamix.com.br:443:127.0.0.1` → HTTP/2 200 servindo `index.html` ✅
7. Validação externa (CC local Mac): `curl --resolve ponto.budamix.com.br:443:187.77.237.231` → HTTPS 200, SSL handshake 31ms ✅

**Estado final:**
- DNS público: ✅ propagado nos resolvers globais
- Certificado SSL: ✅ emitido pra `ponto.budamix.com.br` (Let's Encrypt)
- Servidor Traefik + PM2 `ponto-certo`: ✅ respondendo HTTPS 200
- DNS de provedores brasileiros (Vivo/Claro/NET): ⏳ podem demorar até algumas horas pra atualizar cache. Pedro reportou que ainda não acessava no Chrome local; Playwright também devolveu `ERR_NAME_NOT_RESOLVED` — confirmação de que é cache de resolver upstream do provedor, não problema do servidor.

**Workaround pra desbloquear individualmente:** trocar DNS do dispositivo pra Cloudflare (1.1.1.1) ou Google (8.8.8.8). Pelo 4G/5G do celular costuma propagar mais rápido (DNS da operadora móvel).

**Risco residual baixo:** funcionários que receberem o link `https://ponto.budamix.com.br` via mensagem do agente RH antes do DNS propagar no provedor deles vão ver tela branca. Mitigação: orientar fallback `https://pontocerto.gbformulario.com` enquanto isso (mesmo backend, mesmo banco). Não chegou a ser disparada nesta sessão.
